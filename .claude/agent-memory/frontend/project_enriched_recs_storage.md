---
name: Enriched Recommendations Storage Pattern
description: How enriched recommendation data (rationale, actions, effortDetail) is stored and accessed in the export page
type: project
---

The Prisma `Recommendation` model only stores core fields: category, priority, title, description, impact, effort. There is no `metadata` column on recommendations.

The enriched top-3 recommendation fields (`rationale`, `actions`, `effortDetail`) generated by `enrichTopRecommendations()` in `packages/core/src/report-composer.ts` are intended to be stored in `report.metadata.recommendations` (a `GeneratedRecommendation[]` array alongside `sections` and `coverPage`).

**Why:** The Prisma schema does not have a metadata column on Recommendation, and adding one would require a migration. Storing the full composed output in report metadata keeps the enriched data immutable alongside the report snapshot.

**How to apply:** When reading enriched recs in the export page, read from `meta?.recommendations` first. Fall back to `report.recommendations` (DB shape) for old reports. The `reports.ts` action currently does NOT persist enriched recs to metadata — this is a gap the backend agent needs to close by adding `recommendations: composed.recommendations` to the metadata object when creating a report.
