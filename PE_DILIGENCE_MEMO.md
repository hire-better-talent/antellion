# PE Diligence Memo - Antellion

## 1. Executive Summary

The code shows Antellion is an AI employer-visibility assessment product: it probes LLMs with candidate-intent hiring queries, scores whether a target employer and competitors appear, extracts citations, and produces executive reports. Evidence: product positioning in `README.md:1-3`, architecture/data flow in `docs/architecture.md:5-13`, LLM execution in `apps/jobs/src/llm-client.ts:282-327`, scoring in `packages/core/src/scan-analysis.ts:328-370`, and report composition in `packages/core/src/report-composer.ts:46-54`. Inference: the current business is not yet pure SaaS; it is tech-enabled consulting with productized deliverables, because pricing artifacts define free snapshots, a $14K Baseline Audit, and $75K-$110K+ Phase 2 audits rather than in-app subscription or entitlement flows (`docs/full-assessment-offer-stack.md:22-35`, `docs/full-assessment-offer-stack.md:52-80`, `docs/full-assessment-offer-stack.md:101-150`, `docs/full-assessment-offer-stack.md:202-215`). Preliminary posture: **Watch**. The product thesis is timely and differentiated, but the repo shows demo/early-commercial maturity, no real auth, no in-product billing, thin deployment maturity, single-contributor concentration, and open high-severity dependency advisories.

This is stated in GTM artifacts, not enforced by product code (`docs/gtm-strategy-2026-03-27.md:151-171`). ## 3. Technology Diligence

- **Stack and architecture:** pnpm/Turbo monorepo with Next.js apps, TypeScript packages, Prisma/Postgres, Tailwind UI, and a Node jobs worker (`README.md:5-18`, `package.json:4-23`, `apps/web/package.json:11-30`, `apps/marketing/package.json:11-30`, `apps/jobs/package.json:10-24`, `packages/db/prisma/schema.prisma:1-9`). Architectural pattern: modular monolith plus separate polling worker, not microservices. The architecture doc explicitly says the worker keeps long LLM calls out of Next.js and can scale independently (`docs/architecture.md:32-37`).
- **Code quality signals:** Strong TypeScript posture in shared config (`packages/config/tsconfig.base.json:3-18`) and Zod validation utility (`packages/core/src/validation.ts:31-45`). ESLint exists but `no-explicit-any` is only a warning (`packages/config/eslint.base.mjs:8-14`). Test coverage is concentrated in `packages/core`: 33 test files and 1,309 passing tests from `pnpm test`; apps/jobs/UI have no visible test suites despite significant business logic. Root scripts support build/lint/typecheck/test (`package.json:4-14`), and local `pnpm typecheck` passed all configured package tasks.
- **Scalability posture:** The worker polls every 5 seconds and fetches up to 10 RUNNING scans (`apps/jobs/src/index.ts:15-38`), uses only an in-process `Set` for duplicate suppression (`apps/jobs/src/index.ts:9-12`, `apps/jobs/src/index.ts:44-58`), and executes queries in batches of 5 with a 300ms inter-batch delay (`apps/jobs/src/scan-worker.ts:34-39`, `apps/jobs/src/scan-worker.ts:148-187`). DB uniqueness prevents duplicate scan results (`packages/db/prisma/schema.prisma:276-281`), but there is no durable queue, lease, dead-letter queue, or retry orchestration. Signal scoring uses pairwise duplicate detection and notes O(n^2) behavior, acceptable only because scan sizes are assumed small (`apps/jobs/src/scan-worker.ts:410-418`).
- **Security posture:** Critical gap: auth is a stub that returns the first org in the database (`apps/web/src/lib/auth.ts:3-21`), and dashboard layout explicitly says auth should be added later (`apps/web/src/app/(dashboard)/layout.tsx:3-7`). Server actions often scope by organization, but because org identity is fake, tenant isolation is not production-grade (`apps/web/src/app/(dashboard)/actions/reports.ts:56-67`, `apps/web/src/app/(dashboard)/actions/reports.ts:138-146`). Evidence approval bypasses real actor/self-review controls with `"system"` and `null` placeholders (`apps/web/src/app/(dashboard)/actions/evidence.ts:166-183`, `apps/web/src/app/(dashboard)/actions/evidence.ts:210-255`). Public snapshot links are token-gated and revocable, which is a reasonable pattern but still bearer-token access (`apps/marketing/src/app/s/[token]/page.tsx:50-64`). `.env` is gitignored and `.env.example` is minimal (`.gitignore:5-7`, `.env.example:1-2`).
- **Dependency security:** `pnpm audit --prod` on 2026-04-21 found three high-severity advisories: `next@15.5.14` DoS in Server Components, `effect@3.18.4` AsyncLocalStorage context issue via Prisma, and `defu@6.1.4` prototype pollution via Prisma config. Local lockfile evidence: `pnpm-lock.yaml:1034`, `pnpm-lock.yaml:1065`, `pnpm-lock.yaml:1421`, `pnpm-lock.yaml:2520`, `pnpm-lock.yaml:2541`, `pnpm-lock.yaml:3120`. Patched versions reported by audit: Next >=15.5.15, effect >=3.20.0, defu >=6.1.5.
- **Dependency staleness:** `pnpm outdated --recursive` shows major-line lag on Next 15.5.14 vs 16.2.4, Prisma 6.19.2 vs 7.7.0, Tailwind 3.4.19 vs 4.2.3, Zod 3.25.76 vs 4.3.6, Vitest 3.2.4 vs 4.1.4, TypeScript 5.9.3 vs 6.0.3, and react-markdown 9.1.0 vs 10.1.0. Package manifests specify broad caret ranges (`apps/web/package.json:16-20`, `apps/marketing/package.json:14-19`, `packages/db/package.json:18-25`, `packages/core/package.json:10-17`), while lockfile pins actual resolved versions (`pnpm-lock.yaml:1421`, `pnpm-lock.yaml:1573`, `pnpm-lock.yaml:1600`, `pnpm-lock.yaml:1911`).
- **Technical debt:** TODOs explicitly mark auth gaps and placeholder actors in production-sensitive paths (`apps/web/src/lib/auth.ts:3-8`, `apps/web/src/app/(dashboard)/actions/evidence.ts:166-173`, `apps/web/src/app/(dashboard)/actions/evidence.ts:238-251`, `apps/web/src/app/(dashboard)/actions/qa.ts:256-277`). OpenAI response parsing uses `as any` around Responses API output and web search tool typing (`apps/jobs/src/llm-client.ts:87-98`, `apps/jobs/src/llm-client.ts:109-132`). Legacy support is present in query source and response handling (`packages/db/prisma/schema.prisma:198`, `apps/jobs/src/llm-client.ts:129-132`).
- **Build/deploy maturity:** Vercel-oriented fixes are visible in git history and Prisma binary target (`packages/db/prisma/schema.prisma:1-4`), and Turbo passes `DATABASE_URL` into build tasks (`turbo.json:4-8`). No Dockerfile, Kubernetes, Terraform, or `.github` CI config was found in repo scan. There is only one Prisma migration directory in the repo (`packages/db/prisma/migrations/20260417011231_add_scan_run_share_token/migration.sql`) plus `db:push`, which suggests immature environment promotion discipline (`packages/db/package.json:6-13`).

## 5. Commercial Risk Flags

- **License risk:** No root `LICENSE` file was found, and package manifests omit license fields (`package.json:1-23`, `apps/web/package.json:1-31`, `packages/core/package.json:1-17`). Inference: ownership and third-party distribution rights need legal cleanup before sale. I did not find GPL/AGPL references in package manifests or lockfile search, but `pnpm licenses list --prod --json` failed due a missing pnpm package index file, so license diligence is incomplete.
- **IP risk:** The repo contains third-party PDF assets under `docs/growth-assets` and competitor assets under `docs/competitor-assets`; `.gitignore` specifically marks one competitor PDF as sensitive (`.gitignore:12-13`). Inference: copyrighted sales/growth PDFs and competitor reports in diligence materials may be non-distributable and should be removed or permissioned before data room sharing.
- **Compliance:** DPA and MSA templates exist and are thoughtful, but both say they require licensed attorney review (`docs/dpa-template.md:1-13`, `docs/msa-template.md:1-3`). DPA states processing should be narrow and mostly contact info plus public AI outputs (`docs/dpa-template.md:57-64`). There is no SOC 2 evidence, no access controls, no audit logging tied to real users, no privacy policy code artifact, no data retention implementation, and no HIPAA/PCI posture. Enterprise tier promises SOC2/security review cooperation, not compliance certification (`docs/full-assessment-offer-stack.md:202-215`).
- **Data/privacy:** Lead capture collects company, domain, contact name, email, title, competitor, and primary role (`apps/marketing/src/app/actions.ts:39-48`) and stores it in the `Lead` table (`packages/db/prisma/schema.prisma:528-548`). Slack notifications include contact name and email (`apps/marketing/src/app/actions.ts:127-143`), which is operationally useful but creates PII spread into Slack. Public snapshots are tokenized and `noindex` (`apps/marketing/src/app/s/[token]/page.tsx:38-42`, `apps/marketing/src/app/s/[token]/page.tsx:50-64`), but no expiry, access logging, or rate limiting is evident.
- **Vendor lock-in:** Product depends on OpenAI, Anthropic, Google AI APIs, Vercel/Next, Prisma/Postgres, and Slack (`apps/jobs/package.json:10-18`, `apps/web/package.json:16-20`, `apps/marketing/src/app/actions.ts:124-143`, `packages/db/prisma/schema.prisma:6-9`). Inference: LLM provider diversity reduces single-model risk, but citation methodology is exposed to API/search-tool behavior changes.

## 6. Value-Creation Levers

**Top 3 quick wins, 0-6 months:**

1. Ship real auth, RBAC, and tenant identity. The schema already has organizations, users, and roles (`packages/db/prisma/schema.prisma:55-93`), but auth is stubbed (`apps/web/src/lib/auth.ts:3-21`). This is the gating item for any enterprise pilot.
2. Patch security advisories and stabilize dependency policy. Upgrade Next beyond 15.5.15 and resolve Prisma transitive `effect`/`defu` advisories; lock a monthly `pnpm audit`/`outdated` process. Evidence: audit findings against `pnpm-lock.yaml:1034-1065`, `pnpm-lock.yaml:1421`, `pnpm-lock.yaml:2520-2541`, `pnpm-lock.yaml:3120`.
3. Add CI/CD and deployment evidence. Root scripts exist (`package.json:4-14`) and tests pass, but no CI config was found. Add GitHub Actions for install, typecheck, lint, test, build, Prisma migration validation, and dependency audit.

**Top 3 strategic plays, 6-24 months:**

1. Convert from services-led audits to recurring monitoring SaaS. Schema already supports repeated scan runs and baselines (`packages/db/prisma/schema.prisma:211-243`, `packages/db/prisma/schema.prisma:568-612`), and report code includes multi-run analysis hooks (`apps/web/src/app/(dashboard)/actions/reports.ts:30-32`). Package this into quarterly/continuous monitoring.
2. Build a client portal and executive dashboard. Current public snapshot is token-only and narrow (`apps/marketing/src/app/s/[token]/page.tsx:116-123`); dashboard is internal and unauthenticated (`apps/web/src/app/(dashboard)/layout.tsx:3-7`). A portal could support expansion, renewal, and usage-based pricing.
3. Extend beyond talent into broader AI brand/reputation visibility. Core primitives are query generation, LLM probing, citation extraction, comparison, and report composition (`apps/jobs/src/llm-client.ts:282-327`, `packages/core/src/scan-comparison.ts:31-51`, `packages/core/src/report-composer.ts:46-54`). Inference: architecture can generalize to B2B brand, investor relations, product comparison, or regulated reputation categories.

**Cost-out opportunities:** Replace manual/consulting workflow with durable scan automation and report ops. The worker exists, but no queue/lease system or durable retry framework is present (`apps/jobs/src/index.ts:15-38`, `apps/jobs/src/scan-worker.ts:148-187`). LLM spend controls are not visible; add per-scan budgets, caching of repeated prompts, and provider cost telemetry. Team restructuring cannot be responsibly inferred because the repo only shows one contributor account.

## 7. Deal Breakers & Red Flags

- 🔴 **Critical - No production auth or tenant isolation.** Auth returns the first organization and dashboard layout says login/redirect is future work (`apps/web/src/lib/auth.ts:3-21`, `apps/web/src/app/(dashboard)/layout.tsx:3-7`). This blocks enterprise SaaS deployment.
- 🔴 **Critical - QA/self-review controls are bypassed.** Evidence approval uses `"system"` and `null` actor placeholders, explicitly bypassing self-review until auth exists (`apps/web/src/app/(dashboard)/actions/evidence.ts:166-183`, `apps/web/src/app/(dashboard)/actions/evidence.ts:210-255`).
- 🟡 **Material - Three high-severity dependency advisories.** Audit found high-severity issues in Next, effect, and defu; lockfile shows affected versions (`pnpm-lock.yaml:1034`, `pnpm-lock.yaml:1065`, `pnpm-lock.yaml:1421`, `pnpm-lock.yaml:2520`, `pnpm-lock.yaml:2541`, `pnpm-lock.yaml:3120`).
- 🟡 **Material - Product is services-led, not true SaaS today.** Pricing and Stripe docs describe quotes/invoices and paid audits (`docs/full-assessment-offer-stack.md:52-80`, `docs/stripe-quote-guide.md:7-22`); no billing/entitlement code is present in package manifests or schema (`packages/db/prisma/schema.prisma:528-548`).
- 🟡 **Material - No CI/CD or IaC evidence.** Scripts exist (`package.json:4-14`), but no `.github`, Docker, Terraform, or k8s config was found. This is below institutional software diligence bar.
- 🟡 **Material - Bus factor 1.** All 40 visible commits are from one account in April 2026, with no tags and only `main`/`origin/main`. Inference: execution velocity is real, but institutional continuity is weak.
- 🟡 **Material - PII goes to Slack.** Lead data includes contact name/email/title and is sent to a Slack webhook (`apps/marketing/src/app/actions.ts:39-48`, `apps/marketing/src/app/actions.ts:127-143`) with no visible retention or DSR handling.
- 🟢 **Monitor - Worker scaling is early-stage.** Polling worker plus in-process guard and DB uniqueness works for low volume, but not high-volume multi-tenant scans (`apps/jobs/src/index.ts:15-38`, `apps/jobs/src/index.ts:54-58`, `packages/db/prisma/schema.prisma:276-281`).
- 🟢 **Monitor - Core tests are strong but narrow.** 1,309 core tests passed; app/server-action/worker integration coverage is not evidenced. The root test task only executed `@antellion/core:test` because other packages lack test scripts (`packages/core/package.json:6-17`, `apps/web/package.json:4-10`, `apps/jobs/package.json:4-9`).

## 9. Preliminary Valuation Frame

This should not be valued as mature SaaS unless management proves recurring software revenue, retention, self-serve or repeatable sales motion, and gross margin. Current evidence supports a tech-enabled services frame with an option on vertical SaaS. For SaaS reference, 2026 public/private SaaS sources show broad market ranges around mid-single-digit ARR multiples, with meaningful premiums for high growth, high NRR, and AI-native positioning. External benchmarks used: Multiples.vc April 2026 public software/SaaS comps at https://multiples.vc/reports/software-saas-valuation-multiples, SaaSValuationMultiple April 2026 at https://saasvaluationmultiple.com/, and Acquiry 2026 SaaS multiple commentary at https://www.acquiry.com/saas-valuation-multiples-2026/.

Applicable comps by inference: HR tech / talent intelligence, employer-brand analytics, vertical SaaS, and AI-enabled professional services. Key value drivers: recurring scan revenue, proprietary query/citation methodology, validated ROI on candidate pipeline, enterprise procurement readiness, provider-agnostic LLM evidence capture, and gross margin after automation. Key discounts: no auth, no CI/IaC, single-contributor bus factor, unresolved security advisories, no in-product billing, unknown customer traction, and services-heavy delivery.

## 10. Recommendation

**Final posture: Watch.** Three reasons: first, the product thesis is real and the core analytical engine is more than vaporware, with deterministic scoring, citation analysis, QA checks, and report generation evidenced in code (`packages/core/src/scan-analysis.ts:265-370`, `packages/core/src/scan-comparison.ts:189-215`, `packages/core/src/qa/checks.ts:21-241`, `packages/core/src/report-composer.ts:46-54`). Second, commercial artifacts show a plausible high-ACV wedge into talent leaders (`docs/full-assessment-offer-stack.md:52-80`, `docs/full-assessment-offer-stack.md:101-150`). Third, the current software maturity is too early for a PE control investment: no real auth, no CI/IaC, no SaaS billing, high bus-factor concentration, and open high-severity dependency advisories. The single biggest thing that would change my mind: evidence of repeatable paid customer demand with high gross margin and a shipped auth/tenant-isolated production product.
